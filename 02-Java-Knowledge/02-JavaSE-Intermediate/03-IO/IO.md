### IO 的分类

- 按流分类：输入流、输出流

- 按流中数据的最小单位分类：字节流(适合操作所有类型的文件)、字符流(只适合操作纯文本文件)

  整体来看可分为四类：字节输入流，字节输出流，字符输入流，字符输出流

#### FileOutputStream(字节输出流)

写操作：程序 --> 文件

创建对象时，若文件不存在则会自动创建；如果路径不存在，会抛出FileNotFoundException

**追加开关**：构造方法中路径后传true，否则每次运行都会清空文件内容重写

#### FileInputStream(字节输入流)

读操作：文件 --> 程序

读到文件**末尾**时，返回的是-1

#### FileReader / FileWriter(字符输入流 / 字符输出流)

**本质**：字节流 + 编码表

字符流会自带一个缓冲区（Buffer），如果不关闭流或者不手动 flush()，数据可能还憋在内存里没进硬盘。

#### Properties

本质上是一个Map集合，但它多了一对翅膀：load()从文件读取，store()写回文件



##### *单字节  vs 字节数组*

单字节读写：一个一个搬运

字节数组：一次搬运一组，减少了系统调用的次数。通常定义为byte[] bytes = new byte[1024 * 8]; （8KB是平衡性能和内存的经典比例）

**使用完要使用close()关流**，或者使用try-catch-resources：使用close手动关流需要先开后关，在try-catch-resources中Java会自动按照初始化逆序帮忙关掉。

**字节流读中文会乱码**是因为UTF-8编码下，一个中文占3个字节。如果用1024读取，有可能数组末尾切断了一个中文的3个字节(只读了1个或2个)。剩下的字节在下次读取，这就会乱码。

**IO**类是经典的装饰者模式，IO流的体系非常庞大，不是靠简单的继承，而是靠**装饰者模式**。
